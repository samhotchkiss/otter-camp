Design Principles

Core UX Decisions

  • Operator, not worker: The human provides judgment, agents provide labor. UI optimized for decisions, not task execution.
  • Explicit over inferred: Items only need attention when agents explicitly flag needs_human: true. No guessing.
  • Attention is scarce: Ruthless filtering. Dashboard grouped by urgency. Inbox is action-only. Crankfeed is ambient awareness.
  • Status pulse colors: üî¥ Red = needs you, üü° Yellow = blocked (not on you), üü¢ Green = cranking, ‚ö™ Gray = idle.
  • One-click resolution: Approve/decide/review without leaving context. Big buttons, keyboard shortcuts.
  • Real-time by default: Live updates via WebSocket. No refresh needed. The dashboard breathes.
  • Keyboard power users: j/k navigation, a to approve, g+i for inbox, g+p for projects.

Key Patterns

Reusable Components

  • Project Card: Pulse color, title, activity preview, needs-you badge, agent stack
  • Inbox Item: Type badge, urgency dot, summary, meta, action buttons
  • Task Card: ID, title, agent, priority badge, needs-human border
  • Activity Item: Time, agent avatar, action text, project badge, expandable details
  • Agent Avatar: Colored circle with initial, stackable
  • Status Badges: Approval (red), Decision (yellow), Review (purple), Question (blue)
